PROTOCOLS

Protocols

In networking, a protocol is a set of rules for formatting and processing data. Network protocols are like a common language for computers. The computers within a network may use vastly different software and hardware; however, the use of protocols enables them to communicate with each other regardless.

Protocols make these networking functions possible. For instance, the Internet Protocol (IP) is responsible for routing data by indicating where data packets* come from and what their destination is. IP makes network-to-network communications possible. Hence, IP is considered a network layer (layer 3) protocol.

As another example, the Transmission Control Protocol (TCP) ensures that the transportation of packets of data across networks goes smoothly. Therefore, TCP is considered a transport layer (layer 4) protocol.

HTTP/HTTPS

HTTP/HTTPS - protocols for web communication.

Methods: GET, POST, PUT, DELETE, PATCH.

Response codes: 1xx (info), 2xx (success), 3xx (redirection), 4xx (client error), 5xx (server error).

HTTPS = secure HTTP.

HTTPS is a secure version of HTTP that encrypts data exchanged between clients and servers using SSL/TLS protocols. It uses the same methods and status codes as HTTP but adds an additional layer of security by ensuring data integrity and confidentiality.

DNS

DNS (Domain Name System) is a system that translates domain names into IP addresses, enabling the identification and location of resources on the internet. DNS records are used to store specific types of information associated with a domain. Here are some common types of DNS records:

CNAME (Canonical Name) Record: Creates an alias for a domain name, allowing it to point to another domain name.

MX (Mail Exchanger) Record: Specifies the mail server responsible for accepting incoming email for a domain.

TXT (Text) Record: Allows arbitrary text to be associated with a domain. Commonly used for SPF (Sender Policy Framework) records for email authentication.

NS (Name Server) Record: Specifies the authoritative DNS servers for a domain. Example: example.com -> ns1.example.com

IP - how to determine an IP address

Using Command Prompt/Terminal:

  • Windows, open Command Prompt. On macOS or Linux, open Terminal.
  • Type the command "nslookup" followed by the domain name.
  • Press Enter.
  • The IP address associated with the domain name will be displayed.

Using Online Tools:

  • Various online tools provide IP address lookup services
  • Open a web browser and search for "IP address lookup" or "DNS lookup."
  • Choose a reliable tool from the search results.
  • Enter the domain name in the provided field or option.
  • Initiate the lookup.
  • The resulting page will display the corresponding IP address.
